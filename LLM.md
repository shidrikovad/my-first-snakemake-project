## Примеры работы с LLM

### Пример 1: Помощь с регистрацией

**Мой запрос:**
Почему при регистрации на GitHub не приходит письмо на почту для верификации?

**Ответ LLM (Алиса AI):**

Если при регистрации на GitHub не приходит письмо для верификации, это может быть вызвано несколькими причинами. Вот основные из них и способы решения:
1.	Проверка электронной почты
Убедитесь, что вы правильно ввели адрес электронной почты при регистрации. Даже небольшая опечатка может привести к тому, что письмо не дойдёт. Также проверьте, не заблокирован ли ваш почтовый ящик или не переполнен ли он.
2.	Спам-фильтры и папки
Письмо могло попасть в папку «Спам» или «Нежелательная почта». Проверьте эти разделы в вашем почтовом клиенте. Если письмо там, отметьте его как «Не спам», чтобы в будущем подобные сообщения доставлялись в основной ящик.
3.	Настройки безопасности почтового сервиса
Некоторые почтовые сервисы (например, Gmail, Яндекс.Почта) имеют строгие правила фильтрации писем от незнакомых отправителей. Попробуйте временно отключить эти настройки или добавить домен GitHub (github.com) в список доверенных отправителей.
4.	Проблемы с интернет-соединением или серверами GitHub
Временные сбои в работе серверов GitHub или нестабильное интернет-соединение могут мешать доставке писем. Попробуйте повторить попытку позже или проверить статус сервисов GitHub на официальной странице состояния.
5.	Блокировка писем провайдером или государством
В некоторых регионах доступ к определённым сервисам может быть ограничен. Если вы находитесь в стране с ограничениями интернет-трафика, попробуйте использовать другой почтовый сервис (например, Gmail вместо mail.ru).
6.	Капча и регистрация
Иногда проблемы с прохождением капчи при регистрации могут влиять на отправку писем. Убедитесь, что вы корректно прошли все этапы регистрации, включая проверку капчи. Если возникают трудности с капчей, попробуйте:
 - Отключить блокировщики рекламы и расширения браузера.
 - Использовать другой браузер или устройство.
 - Очистить кеш и куки.
 - Включить режим инкогнито.
7.	Проверка почтового сервиса
Некоторые почтовые сервисы могут блокировать письма от GitHub. Попробуйте зарегистрироваться, используя адрес электронной почты от другого провайдера (например, Gmail, Yahoo), а затем добавить основной адрес как резервный в настройках GitHub.
Если проблема сохраняется, рекомендуется:
 - Обратиться в службу поддержки GitHub через официальный канал связи.
 - Проверить документацию GitHub на предмет актуальных инструкций по регистрации и верификации.
 - Убедиться, что ваш почтовый сервис поддерживает получение писем от внешних отправителей.
Если вы используете корпоративную или университетскую почту, уточните у администратора, нет ли ограничений на получение писем от GitHub.

### Пример 2: Проверка кода

**Мой запрос:**

Проверь код на наличие ошибок

**Ответ LLM (DeepSeek):**

Код работает корректно, ошибок нет. Есть одно логическое несоответствие:
Вы фильтруете данные по qsec < 18, но затем создаёте категорию performance с условием qsec >= 17.5 ~ "comfort". При фильтре qsec < 18 категория "comfort" никогда не появится — в неё попадают значения от 17.5 до 17.999..., но фильтр их уже отсек.
Остальной код (фильтрация, группировка, вывод) — в полном порядке.
